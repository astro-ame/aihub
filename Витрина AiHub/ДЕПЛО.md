# Деплой бота Ai Hub Market на сервер

## В панели сервера (при создании VDS)

1. **Cloud-init** — вставь содержимое файла `cloud-init.sh` в поле сценария (или нажми «Загрузить скрипт из файла» и выбери `cloud-init.sh`).
2. Создай сервер и дождись первой загрузки.

---

## В консоли (подключись по SSH к серверу)

### 1. Подключение
```bash
ssh root@IP_АДРЕС_СЕРВЕРА
```
(или `ssh ubuntu@IP_АДРЕС_СЕРВЕРА` — смотря какой пользователь в панели)

---

### 2. Загрузка файлов бота на сервер

**Вариант А — с твоего компьютера (в своём терминале, не на сервере):**
```bash
cd /Users/artem/Downloads/Работа/TG_BOTS/Витрина\ AiHub
scp -r bot.py config.py database.py admin_handlers.py manager_handlers.py utils_dt.py requirements.txt data root@IP_АДРЕС_СЕРВЕРА:/opt/aihub-bot/
```
Если есть `pally_client.py` и папка `scripts` — добавь их в список через пробел.

**Вариант Б — если файлы уже в архиве:** залей архив через панель (Файловый менеджер / SFTP), затем на сервере:
```bash
cd /opt/aihub-bot
unzip -o aihub-bot.zip
```

---

### 3. На сервере: Python, venv и зависимости
```bash
cd /opt/aihub-bot
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

---

### 4. На сервере: файл .env
```bash
nano /opt/aihub-bot/.env
```
Вставь свои переменные (BOT_TOKEN, ADMIN_IDS, MANAGER_IDS, пароли, ID групп и т.д.), сохрани: `Ctrl+O`, Enter, выход: `Ctrl+X`.

Либо скопируй .env с компьютера:
```bash
scp .env root@IP_АДРЕС_СЕРВЕРА:/opt/aihub-bot/
```
(команду выполнить на своём компьютере из папки проекта)

---

### 5. На сервере: запуск бота
```bash
systemctl enable aihub-bot
systemctl start aihub-bot
systemctl status aihub-bot
```

---

### 6. Полезные команды на сервере
```bash
# Смотреть логи в реальном времени
journalctl -u aihub-bot -f

# Перезапустить бота
systemctl restart aihub-bot

# Остановить бота
systemctl stop aihub-bot
```

---

## Обновление бота (когда он уже работает)

Если бот уже развёрнут и нужно просто залить изменения после правок в коде:

**1. На Mac (в терминале) — залить изменённые файлы:**
```bash
cd "/Users/artem/Downloads/Работа/TG_BOTS/Витрина AiHub"
scp bot.py config.py admin_handlers.py manager_handlers.py root@5.42.97.186:/opt/aihub-bot/
```
Если менял только часть файлов — перечисли в `scp` только их (например, только `config.py`).

**2. На сервере — перезапустить бота:**
```bash
ssh root@5.42.97.186 "systemctl restart aihub-bot"
```

**3. (По желанию) Проверить логи:**
```bash
ssh root@5.42.97.186 "journalctl -u aihub-bot -n 30 --no-pager"
```

После деплоя текущих изменений (канал, группы заказов/уведомлений) достаточно залить `config.py`, `bot.py`, `admin_handlers.py`, `manager_handlers.py` и выполнить перезапуск.

## Кратко

| Где | Что делать |
|-----|------------|
| **Панель сервера** | При создании VDS — вставить/загрузить скрипт cloud-init. |
